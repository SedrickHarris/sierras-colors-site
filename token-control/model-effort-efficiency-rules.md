# Model and Effort Efficiency Rules

Use these rules to select the right model and reasoning effort.

## Core Rule

Use the lowest effective model and effort level for the task.

## Use Low Effort For

- Folder/file creation
- Markdown updates
- Basic checklists
- Simple formatting
- Short commands
- Simple confirmations

Token goal:
Minimum usage.

## Use Medium Effort For

- Standard keyword maps
- Standard page outlines
- Standard schema planning
- Developer build briefs
- Claude Code prompts
- Normal internal linking
- Basic content edits

Token goal:
Balanced quality and efficiency.

## Use High Effort For

- 10-metric analysis
- Visibility alignment
- Competitive analysis
- Entity mapping
- Conversion optimization
- AI citation readiness
- Content moat planning
- Final QA
- Prompt evaluation

Token goal:
Spend more only when judgment matters.

## Use Strongest Effort For

- Complex architecture
- Revenue-critical pages
- Competitive homepage strategy
- Broken build debugging
- Major refactors
- Deployment failures
- Prompt system redesign
- Benchmark comparison

Token goal:
Use only when mistakes are costly.

## Escalation Triggers

Escalate if:

- Output is generic
- Output misses the page goal
- Page is high value
- Page is revenue-critical
- Build breaks
- Same issue repeats
- QA finds critical blockers
- Prompt output scores below target

## De-Escalation Triggers

De-escalate if:

- Task is repetitive
- Output only needs formatting
- Strategy is approved
- Claude Code only needs a brief
- Task is not competitive
- User asks for speed

## Final Rule

Do not pay for deep reasoning when a structured template can solve the task.
