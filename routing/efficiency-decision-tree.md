# Site OS Efficiency Decision Tree

Use this decision tree before starting any major Site OS task.

## Step 1: What is the task?

If the task is file setup, formatting, or folder creation:
Use low effort and compact output.

If the task is a normal website strategy task:
Use Core Mode and medium effort.

If the task is a high-value page strategy:
Use Beyond-Elite Mode and high effort.

If the task is revenue-critical or highly competitive:
Use Full Competitive Build Mode and high or strongest effort.

If the task is debugging:
Use medium effort first, then escalate if the issue repeats.

## Step 2: Is the strategy already approved?

If yes:
Do not rerun strategy prompts.
Compress context and create the next handoff.

If no:
Run the smallest strategy workflow needed.

## Step 3: Is this page high value?

If no:
Use Fast or Core Mode.

If yes:
Use Beyond-Elite Mode.

If revenue-critical:
Use Full Competitive Build Mode.

## Step 4: Is competition high?

If no or unknown:
Do not automatically use Full Competitive Build Mode.

If yes:
Use SERP analysis, page moat, and AI citation prompts.

## Step 5: Does Claude Code need full strategy?

If no:
Send a compact build brief.

If yes:
Send only the approved strategy summary, not the full conversation.

## Step 6: What output depth is needed?

If user needs action:
Compact.

If user needs normal planning:
Standard.

If user needs evaluation or strategy:
Deep.

## Step 7: Should the task stop?

Stop when:

- The output is actionable
- The next step is clear
- More detail would not improve implementation
- The task has enough information for Claude Code
- The page can move to QA

## Final Decision Format

Use this format:

Recommended mode:
[Fast/Core/Beyond-Elite/Full Competitive]

Recommended effort:
[Low/Medium/High/Strongest]

Recommended prompt:
[Prompt file]

Output depth:
[Compact/Standard/Deep]

Reason:
[Short explanation]

Token-saving move:
[What to skip or summarize]

Next step:
[Next action]
