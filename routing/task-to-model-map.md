# Task to Model and Effort Map

Use this file to choose the right model and effort level for Site OS tasks.

## Low Effort

Use for:

- File organization
- Folder creation
- Markdown formatting
- Simple checklist updates
- Basic metadata edits
- Naming conventions
- Simple content cleanup
- Copying templates

Why:
These tasks are low complexity and do not require deep reasoning.

## Medium Effort

Use for:

- Keyword grouping
- Standard page outlines
- AEO FAQ planning
- Internal linking strategy for normal pages
- Schema planning with clear data
- Developer build briefs
- Claude Code prompt creation
- Standard content edits
- Normal page implementation planning

Why:
These tasks need structure, but not maximum reasoning.

## High Effort

Use for:

- 10-metric analysis
- Full visibility and conversion alignment
- SERP competitive analysis
- Entity and topical authority mapping
- Conversion optimization
- AI citation readiness
- Page moat strategy
- Final QA
- Prompt evaluation

Why:
These tasks require scoring, prioritization, judgment, and risk control.

## Strongest Model / Advisor / Ultrathink

Use for:

- Complex site architecture
- Homepage strategy
- Core money pages
- Highly competitive local SEO pages
- Debugging broken builds
- Major refactors
- Deployment issues
- Prompt system redesign
- Benchmark and eval comparison

Why:
These tasks are high impact, high complexity, or high risk.

## Escalation Triggers

Escalate if:

- Output is too generic
- Page is high value
- Page is highly competitive
- Build fails
- Code errors repeat
- Strategy affects many pages
- Final launch decision is needed
- User asks for strict grading

## De-Escalation Triggers

Use lower effort if:

- Task is formatting
- Task is repetitive
- Structure is already approved
- User only needs a short prompt
- Page is low priority
- Speed matters
