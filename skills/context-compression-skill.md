# Context Compression Skill

Use this skill to reduce token usage before moving from one Site OS step to another.

The goal is to preserve the important strategy while removing unnecessary detail.

## When to Use

Use before:

- Creating a developer build brief
- Creating a Claude Code prompt
- Moving from strategy to implementation
- Continuing a long workflow
- Starting a new page using previous decisions
- Uploading or pasting context into Claude Code

## Compression Rule

Do not paste the entire conversation when a summary will work.

Compress approved strategy into a short, structured brief.

## What to Keep

Keep:

- Business name
- Page name
- Page type
- Route
- Primary keyword
- Target location if applicable
- Target audience
- Primary CTA
- Approved page sections
- Metadata
- Schema requirements
- Internal links
- Image requirements
- Conversion requirements
- Important guardrails
- QA requirements

## What to Remove

Remove:

- Repeated explanations
- Earlier rejected options
- Long reasoning
- Duplicate scorecards
- Old versions
- Generic SEO advice
- Full benchmark notes unless needed
- Full prompt text unless needed

Compress context before handoff.
