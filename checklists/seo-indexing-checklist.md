# SEO Indexing Checklist

Use this checklist after publishing a page and before/after submitting it to indexing tools.

## Purpose

This checklist helps prepare pages for Google Search Console, Bing Webmaster Tools, indexing, crawlability, local visibility, and search performance monitoring.

## 1. Pre-Indexing Checks

- [ ] Final page URL is confirmed
- [ ] Page loads successfully
- [ ] Page is not accidentally noindexed
- [ ] Page is not blocked by robots.txt
- [ ] Canonical is correct if applicable
- [ ] Page is linked internally
- [ ] Metadata is present
- [ ] H1 is present
- [ ] Content is not thin
- [ ] Duplicate content risk is low
- [ ] Schema matches visible content

## 2. Google Search Console

- [ ] Property is connected
- [ ] Sitemap is submitted if applicable
- [ ] URL is inspected
- [ ] Page is requested for indexing if appropriate
- [ ] Coverage/indexing status is monitored
- [ ] Mobile usability is checked
- [ ] Enhancements/rich results are monitored if applicable
- [ ] Query performance is reviewed after indexing

## 3. Bing Webmaster Tools

- [ ] Site is connected
- [ ] Sitemap is submitted if applicable
- [ ] URL submission is completed if appropriate
- [ ] Crawl status is checked
- [ ] Indexing status is monitored
- [ ] Search performance is reviewed after indexing

## 4. Local Platform Checks

If the page supports local intent:

- [ ] Google Business Profile alignment is checked
- [ ] Bing Places alignment is checked
- [ ] Apple Business Connect / Apple Maps alignment is checked where applicable
- [ ] Service area language is consistent
- [ ] Business name is consistent
- [ ] Contact path is consistent
- [ ] No conflicting local business details exist

If not local:

- [ ] Local platform checks are not applicable

## 5. Rich Result Checks

- [ ] FAQ schema is valid if used
- [ ] Breadcrumb schema is valid if used
- [ ] Service schema is valid if used
- [ ] LocalBusiness schema is valid if used
- [ ] Organization schema is valid if used
- [ ] Article schema is valid if used
- [ ] Schema does not include fake reviews, ratings, prices, hours, or unsupported facts

## 6. Internal Linking After Publish

- [ ] Page is linked from relevant parent pages
- [ ] Page is linked from related service pages
- [ ] Page is linked from related location pages if applicable
- [ ] Page is linked from related blog posts if useful
- [ ] Page links outward to relevant internal pages
- [ ] Anchor text is natural
- [ ] Page is not orphaned

## 7. Post-Indexing Monitoring

Monitor:

- [ ] Indexing status
- [ ] Impressions
- [ ] Clicks
- [ ] Average position
- [ ] Queries
- [ ] CTR
- [ ] Rich result enhancements
- [ ] Page experience issues
- [ ] Mobile usability
- [ ] Bing search performance
- [ ] Local visibility if applicable

## 8. Optimization Triggers

Revisit the page if:

- [ ] Page is not indexed after a reasonable period
- [ ] Impressions are low
- [ ] CTR is low
- [ ] Ranking is poor
- [ ] Queries do not match the target intent
- [ ] Competitors outperform the page
- [ ] Rich results do not appear
- [ ] AI/LLM answers do not reflect the content
- [ ] Conversion is weak
- [ ] Local visibility is weak

## 9. Improvement Actions

Possible improvements:

- [ ] Improve meta title
- [ ] Improve meta description
- [ ] Add better direct-answer section
- [ ] Improve FAQs
- [ ] Add internal links
- [ ] Strengthen topical authority
- [ ] Add supporting blog content
- [ ] Improve local specificity
- [ ] Improve schema accuracy
- [ ] Improve CTA path
- [ ] Improve trust signals
- [ ] Refresh content
