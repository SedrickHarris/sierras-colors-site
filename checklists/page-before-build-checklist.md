# Page Before Build Checklist

Use this checklist before creating a developer build brief or sending a prompt to Claude Code.

## Purpose

This checklist confirms the page strategy is ready before implementation begins.

Do not build until the required strategy, content structure, metadata, schema needs, internal links, and conversion path are clear.

## 1. Page Strategy

- [ ] Business name is confirmed
- [ ] Website name is confirmed
- [ ] Page name is confirmed
- [ ] Page type is confirmed
- [ ] Page route / URL slug is confirmed
- [ ] Target audience is confirmed
- [ ] Primary CTA is confirmed
- [ ] Secondary CTA is confirmed if needed
- [ ] Page goal is clear
- [ ] Search intent is clear
- [ ] Page priority is identified: Low, Medium, High, or Revenue-critical
- [ ] Workflow mode is selected: Fast, Core, Beyond-Elite, or Full Competitive Build

## 2. Keyword and Search Intent

- [ ] Primary keyword is selected
- [ ] Secondary keywords are selected
- [ ] Long-tail keywords are included where useful
- [ ] Local keywords are included if applicable
- [ ] AEO question keywords are included
- [ ] Voice search phrases are included
- [ ] Transactional keywords are identified
- [ ] Informational keywords are identified
- [ ] Search intent matches the page type
- [ ] Keywords are mapped to sections naturally

## 3. Multi-Platform Search Structure

The page strategy should support:

- [ ] Google Search
- [ ] Bing
- [ ] Apple Search / Apple Maps if local
- [ ] Google Business Profile if local
- [ ] AI assistants
- [ ] ChatGPT
- [ ] Perplexity
- [ ] Featured snippets
- [ ] People Also Ask
- [ ] Rich snippets
- [ ] Topical authority
- [ ] Lead generation

## 4. Content Structure

- [ ] H1 is planned
- [ ] Quick answer section is planned
- [ ] Service, topic, or category definition is planned
- [ ] Main content sections are planned
- [ ] Problem or pain point section is planned
- [ ] Process or how-it-works section is planned where useful
- [ ] Trust section is planned
- [ ] Local relevance section is planned if applicable
- [ ] Related services or internal links section is planned
- [ ] AEO FAQ section is planned
- [ ] Final CTA section is planned

## 5. AEO, Featured Snippet, and PAA Readiness

- [ ] Main question is answered near the top
- [ ] Direct answer statements are planned
- [ ] Question-based headings are planned where natural
- [ ] Featured snippet opportunities are identified
- [ ] People Also Ask style questions are included
- [ ] FAQ answers are concise and useful
- [ ] Voice search phrasing is natural
- [ ] AI citation-worthy statements are planned

## 6. GEO and Local Readiness

If local intent applies:

- [ ] Target location is included naturally
- [ ] Nearby areas are identified where useful
- [ ] Local customer problems are included
- [ ] Local service needs are included
- [ ] Local trust signals are included
- [ ] Google Business Profile relevance is considered
- [ ] Bing Places relevance is considered
- [ ] Apple Maps / Apple Search relevance is considered
- [ ] Duplicate location page risk is checked
- [ ] City-name swapping is avoided

If local intent does not apply:

- [ ] GEO/local requirements are marked not applicable

## 7. Topical Authority

- [ ] Core entities are identified
- [ ] Supporting entities are identified
- [ ] Related services are identified
- [ ] Related locations are identified if applicable
- [ ] Supporting blog topics are identified if useful
- [ ] Internal links are planned
- [ ] Page supports the larger site architecture
- [ ] Page does not become an orphan page

## 8. Schema Planning

Recommended schema is identified:

- [ ] WebPage
- [ ] Organization if applicable
- [ ] LocalBusiness if applicable
- [ ] Service if applicable
- [ ] FAQPage if FAQs are visible
- [ ] BreadcrumbList if page hierarchy exists
- [ ] Article or BlogPosting if blog/article

Schema guardrails:

- [ ] No fake reviews
- [ ] No fake ratings
- [ ] No fake pricing
- [ ] No fake address
- [ ] No fake hours
- [ ] No unsupported credentials
- [ ] Schema matches visible page content

## 9. Internal Linking

- [ ] Outbound internal links are planned
- [ ] Inbound internal link opportunities are planned
- [ ] Anchor text is natural
- [ ] Links support user journey
- [ ] Links support topical authority
- [ ] Links support conversion where appropriate
- [ ] Too many links are avoided
- [ ] Irrelevant links are avoided

## 10. Conversion Strategy

- [ ] Above-the-fold CTA is planned
- [ ] Mid-page CTA is planned if appropriate
- [ ] Final CTA is planned
- [ ] Primary conversion path is clear
- [ ] Secondary conversion path is clear if needed
- [ ] Trust signals are planned
- [ ] Objections are identified
- [ ] Mobile CTA visibility is considered
- [ ] Lead capture path is clear

## 11. Claude Code Readiness

- [ ] Developer build brief is ready or planned
- [ ] Approved sections are clear
- [ ] Metadata is included
- [ ] Schema requirements are included
- [ ] Internal links are included
- [ ] Image requirements are included
- [ ] Guardrails are included
- [ ] QA expectations are included
- [ ] Files likely to inspect are noted if known
- [ ] Files likely to create/update are noted if known

## 12. Final Before-Build Decision

Choose one:

- [ ] Ready for developer build brief
- [ ] Ready for Claude Code build prompt
- [ ] Needs minor strategy refinement
- [ ] Needs major strategy revision
- [ ] Not ready to build
