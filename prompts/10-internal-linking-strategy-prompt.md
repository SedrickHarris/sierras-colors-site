# Beyond Elite Internal Linking Strategy Prompt

You are acting as a senior SEO strategist, topical authority planner, UX strategist, conversion strategist, and site architecture specialist.

Your job is to create an internal linking strategy that supports search visibility, topical authority, crawlability, user journey, engagement, and conversion.

Do not create random internal links.

Every internal link must have a clear purpose.

Do not force a fixed number of links. Recommend the best number of links based on page type, page length, site architecture, content depth, and conversion goal.

Prioritize link quality over link quantity.

## Objective

Create an internal linking strategy that supports:

1. SEO
2. AEO
3. Topical authority
4. Crawlability
5. User journey
6. Conversion
7. Local relevance
8. Search dominance

## Part 1: Recommended Link Volume

Recommend the ideal number of internal links for this specific page.

Base the recommendation on:

- Page type
- Page length
- Search intent
- Site architecture
- Available related pages
- Conversion goal
- User journey
- Topical authority opportunity

Use these as flexible guidance, not hard rules:

- Homepage: usually 10 to 25 internal links
- Core service page: usually 5 to 10 internal links
- Location page: usually 5 to 12 internal links
- Blog post: usually 4 to 8 internal links
- Landing page: usually 2 to 5 internal links
- Pillar or category page: usually 12 to 30 internal links

Then state:

- Recommended outbound link count
- Recommended inbound link count
- Why this amount is appropriate
- When to use fewer links
- When to use more links

## Part 2: Outbound Internal Links From This Page

Recommend the strongest outbound links from this page.

Do not force a fixed number.

For each link, include:

- Target page
- Suggested anchor text
- Link placement
- Link status: Required, Recommended, or Optional
- User journey purpose
- Topical authority purpose
- Conversion purpose
- Priority level
- Notes for Claude Code

Only include links that support at least one of the following:

- SEO/topical authority
- User journey
- Conversion
- Local relevance
- Service relationship
- Supporting education

## Part 3: Inbound Internal Links To This Page

Recommend pages that should link back to this page.

Do not force a fixed number.

For each link, include:

- Source page
- Suggested anchor text
- Suggested placement
- Link status: Required, Recommended, or Optional
- Why the link helps SEO
- Why the link helps users
- Priority level

## Part 4: Anchor Text Strategy

Create anchor text recommendations.

Include:

- Exact-match anchors to use sparingly
- Partial-match anchors
- Natural language anchors
- Branded anchors
- CTA-style anchors
- Local anchors if applicable
- Anchors to avoid

Explain how to keep anchor text natural and avoid over-optimization.

## Part 5: Topical Authority Link Map

Explain how the internal links support topical authority.

Include:

- Pillar page connections
- Cluster page connections
- Service page connections
- Location page connections
- Blog support connections
- Related service connections
- Conversion page connections

## Part 6: User Journey Link Map

Explain how links help users move through the site.

Include:

- Awareness-stage links
- Consideration-stage links
- Decision-stage links
- Conversion-stage links
- Support or FAQ links

## Part 7: Link Placement Strategy

Recommend where links should appear.

Options include:

- Hero section
- Quick answer section
- Service overview
- Problem section
- Process section
- Related services section
- Local relevance section
- FAQ answers
- CTA section
- Footer or navigation if applicable

For each placement, explain the purpose.

## Part 8: Link Pruning and Quality Control

Review the recommended links and remove weak links.

A link should be removed or marked optional if it:

- Does not help the user
- Does not support topical authority
- Distracts from the primary CTA
- Uses awkward anchor text
- Creates over-optimization risk
- Repeats the same destination too many times
- Points to a page that is not directly relevant
- Makes the page feel cluttered

Return:

- Links to keep
- Links to remove
- Links to make optional
- Reason for each decision

## Part 9: Internal Linking Risk Check

Check for:

- Too many links
- Too few links
- Irrelevant links
- Repetitive anchor text
- Over-optimized anchors
- Links that distract from conversion
- Broken user journey
- Missing link back to important pages
- Orphan page risk
- Duplicate links with no added value

For each risk, provide a prevention strategy.

## Part 10: Final Internal Linking Plan

End with:

1. Recommended total outbound links
2. Recommended total inbound link opportunities
3. Required outbound links
4. Recommended outbound links
5. Optional outbound links
6. Required inbound link opportunities
7. Recommended inbound link opportunities
8. Best anchor text suggestions
9. Links to avoid
10. Claude Code implementation notes
11. Final internal linking priority score

Do not write full page content.

Focus on link strategy, site architecture, user journey, topical authority, conversion, and implementation readiness.
