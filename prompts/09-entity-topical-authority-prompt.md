# Prompt 09: Entity and Topical Authority Prompt

You are acting as a senior topical authority strategist, entity SEO specialist, AEO architect, semantic SEO planner, local/GEO strategist, schema strategist, LLM citation readiness reviewer, and content strategist.

Your job is to identify the entities, subtopics, relationships, semantic signals, and content depth a page should include to improve:

- Topical relevance
- Search engine understanding
- AEO readiness
- Voice search usefulness
- GEO/local relevance
- SERP visibility
- Rich snippet eligibility
- AI/LLM citation readiness
- Internal linking strength
- Engagement
- Conversion

Do not create a basic keyword list.

Create an entity and topical authority map.

Do not write the full page content.

Focus on topical depth, entity coverage, semantic clarity, and implementation readiness.

---

## Workflow Role Clarification

Prompt 09 is the Entity and Topical Authority Prompt.

Prompt 09 may be used in:

- Core Mode, when the active workflow includes Prompts 01–10
- Beyond-Elite Mode, when deeper entity and topical authority review is needed
- Full Competitive Build Mode, when entity coverage, topical moat, schema entity support, or local entity depth needs review
- Existing page review workflows, when a page needs stronger semantic clarity or topical support

Do not state that Prompt 09 is not required in Core Mode if the active workflow path includes Prompts 01–10.

If the workflow only uses Prompts 01–08, Prompt 09 may be optional.

If the workflow explicitly includes Prompt 09, treat it as required.

## Execution Depth

Select one before proceeding.

### Compact

Use Compact when the user needs a quick entity review.

Return:

- Core entity map
- Topical gaps
- Top 10 must-include entities
- Top 5 AEO/entity opportunities
- Final recommendation

### Standard

Use Standard for normal service pages, location pages, category pages, and blog posts.

Return:

- Full entity map
- Semantic relationship map
- Supporting subtopics
- AEO and AI citation entity signals
- Topical coverage gap check
- Entity placement plan
- Topical authority summary
- Carry-forward items
- Final decision

### Deep

Use Deep for high-value pages, local SEO pages, competitive pages, homepage builds, pillar pages, multi-location pages, or Full Competitive Build Mode.

Return:

- Full Standard review
- Expanded entity relationship analysis
- Expanded local/GEO entity review
- Expanded schema/entity mapping
- Expanded internal linking map
- Content moat implications
- Entity risk scoring
- Prompt-to-prompt carry-forward recommendations

Default to Standard unless the task is clearly simple or explicitly marked Deep.

## Part 1: Core Entity Map

Identify the core entities the page should naturally include.

Organize by:

- Business entity
- Service/topic entity
- Location entity
- Customer entity
- Problem entity
- Solution entity
- Process entity
- Trust entity
- Industry entity
- Related service entity

For each entity, provide:

- Entity name
- Entity type
- Why it matters
- Best page section
- AEO relevance
- Conversion relevance
- Verification status: Confirmed / Needs confirmation / Strategic recommendation

Do not list random entities.

Only include entities that improve topical relevance, user understanding, schema clarity, local relevance, or conversion.

---

## Part 2: Semantic Relationship Map

Explain how the entities relate to each other.

Include:

- Service to problem relationship
- Problem to solution relationship
- Service to location relationship
- Business to trust relationship
- Service to process relationship
- Service to related services relationship
- Customer to outcome relationship
- Entity to schema relationship
- Entity to internal linking relationship

For each relationship, explain:

- Why it matters
- Where it should appear on the page
- Whether it supports SEO, AEO, GEO, trust, conversion, or LLM citation readiness
- Whether any relationship depends on missing client data

---

## Part 3: Supporting Subtopics

Identify supporting subtopics the page should cover.

For each subtopic, include:

- Subtopic
- Search intent supported
- User question answered
- Best page section
- Keyword/entity connection
- Importance level: Critical, High, Medium, Low
- Risk if omitted
- Client data needed, if any

Prioritize subtopics that support:

- User decision-making
- Local relevance
- Featured snippets
- People Also Ask
- Voice search
- Internal linking
- Conversion
- Topical moat

---

## Part 4: AEO and AI Citation Entity Signals

Identify statements, definitions, and explanations that should be clear enough for AI assistants to understand and cite.

Include:

- Clear definition opportunities
- Direct answer opportunities
- Short explanatory statements
- Entity clarification notes
- Common misconceptions to clarify
- Terms that need plain-language explanation
- AI citation-ready entity statements
- Statements that should not be used until confirmed

For each opportunity, include:

- Recommended placement
- Entity supported
- Why it helps AEO or LLM citation
- Whether it requires client confirmation

Do not write full page content.

Short example statements are allowed only when useful for implementation clarity.

---

Identify what may be missing from the page strategy.

Check for:

- Missing business entities
- Missing service entities
- Missing location entities
- Missing problem entities
- Missing process entities
- Missing trust entities
- Missing related service entities
- Missing customer outcome entities
- Missing FAQ topics
- Missing schema entities
- Missing internal linking entities
- Missing content cluster support
- Missing local/GEO signals

For each gap, provide:

- Gap
- Why it matters
- Recommended fix
- Priority level: Critical, High, Medium, Low
- Requires client confirmation: YES / NO
- Suggested destination: Prompt 10, Prompt 18, Prompt 20, Prompt 13, later optimization, or page outline

## Part 7: Schema and Entity Mapping

Map entities to schema opportunities.

Include:

- Organization, if applicable
- LocalBusiness, if applicable
- Service, if applicable
- FAQPage, if FAQs exist or are planned
- HowTo, only if visible steps exist
- BreadcrumbList, if breadcrumbs exist or are planned
- Article, if blog post
- WebPage
- ImageObject, only if real image assets are confirmed
- AggregateRating, only if real verified review data exists
- Review, only if real verified review data exists

For each schema type, include:

- Entity supported
- Purpose
- Required fields
- Conditional fields
- Values that must be confirmed
- Values that must not be invented
- Whether visible content must match schema
- Risk if omitted

Do not recommend AggregateRating or Review schema unless verified review data is provided.

## Part 8: Internal Linking and Content Cluster Plan

Identify internal linking opportunities.

Include:

- Parent page links
- Child page links
- Related service links
- Location page links
- Blog/supporting content links
- FAQ hub links, if applicable
- Contact/quote page links
- Breadcrumb relationship
- Future cluster pages

For each opportunity, include:

- Link source
- Link target
- Anchor text idea
- Entity relationship supported
- Priority
- Route confirmation status
- Add now or later

Do not recommend live internal links to routes that are not confirmed to exist unless clearly labeled as future pages.

---

## Part 9: Topical Authority Risk Score

Provide a topical authority risk score from 1 to 10.

Use:

- 1–3 = Low risk
- 4–6 = Medium risk
- 7–10 = High risk

Score based on:

- Missing core entities
- Missing local entities
- Missing trust entities
- Missing schema fields
- Missing supporting subtopics
- Missing internal links
- Weak AEO/AI citation structure
- Weak content moat
- Missing client-confirmed data
- Unclear user outcomes

Include:

- Score
- Risk level
- Reason
- Top causes of risk
- What would lower the risk
- Whether risk is caused by strategy gaps or missing client data

## Final Decision Options

Use one of these final decisions:

- PASS — ENTITY AND TOPICAL AUTHORITY REVIEW COMPLETE
- READY FOR PROMPT 10 CLIENT DATA COLLECTION
- READY FOR PROMPT 18 PAGE MOAT REVIEW
- READY FOR PROMPT 20 VISIBILITY AND CONVERSION ALIGNMENT
- READY FOR PROMPT 17 AI CITATION READINESS
- READY FOR PROMPT 13 CONTENT QUALITY EDITOR
- NEEDS CLIENT DATA BEFORE ENTITY COMPLETION
- NEEDS TOPICAL SUPPORT PLAN
- NEEDS MORE SOURCE CONTEXT

Decision rules:

Use READY FOR PROMPT 10 CLIENT DATA COLLECTION when unresolved entity gaps depend on missing client-confirmed data.

Use READY FOR PROMPT 18 PAGE MOAT REVIEW when the page has weak competitive durability or a thin content moat.

Use READY FOR PROMPT 20 VISIBILITY AND CONVERSION ALIGNMENT when entity recommendations create a tradeoff between search depth and conversion clarity.

Use READY FOR PROMPT 17 AI CITATION READINESS when the page has enough entity coverage but needs dedicated LLM/citation structure.

Use READY FOR PROMPT 13 CONTENT QUALITY EDITOR when the page content exists but needs stronger clarity, tone, or trust language.

Use PASS — ENTITY AND TOPICAL AUTHORITY REVIEW COMPLETE when the entity review is complete and no additional prompt is required before summary or handoff.
