## Objective

Create a schema strategy that supports:

1. Technical SEO
2. Rich snippets
3. AEO
4. Voice search
5. SERP visibility
6. AI/LLM citation readiness
7. Local SEO if applicable
8. Content clarity

## Part 1: Recommended Schema Types

Recommend applicable schema types.

Possible types:

- WebPage
- Organization
- LocalBusiness
- Service
- FAQPage
- BreadcrumbList
- Article
- BlogPosting
- Product
- SoftwareApplication
- Review, only if real review data is provided
- AggregateRating, only if real rating data is provided

For each schema type, include:

- Schema type
- Why it applies
- Required fields
- Optional fields
- Data needed
- Risk level
- Implementation priority

## Part 2: Schema Field Plan

For each recommended schema type, provide a field plan.

Include:

- Field name
- Recommended value
- Source of truth
- Required or optional
- Notes
- Do not use if data is unavailable

## Part 3: FAQ Schema Strategy

If FAQs are included, provide:

- FAQ questions that should be marked up
- FAQ questions that should not be marked up
- Reason for inclusion or exclusion
- Direct-answer style notes
- Risks to avoid

## Part 4: Breadcrumb Schema Strategy

Provide:

- Recommended breadcrumb path
- Breadcrumb names
- Breadcrumb URLs
- Notes for implementation

## Part 5: Local Schema Strategy

If the page is local, recommend:

- LocalBusiness use
- Service area use
- Address handling
- Phone handling
- Geo handling only if exact data is available
- Opening hours only if verified
- SameAs links only if provided
- Risks to avoid

If local schema is not applicable, state why.

## Part 6: Service Schema Strategy

If this is a service page, recommend:

- Service name
- Service type
- Provider
- Area served
- Description
- Related services
- Offers only if real offer/pricing data is provided

## Part 7: Article or Blog Schema Strategy

If this is a blog post or article, recommend:

- Article headline
- Description
- Author
- Publisher
- Date published if available
- Date modified if available
- Image if available
- Main entity of page

## Part 8: Schema Risk Check

Check for:

- Fake reviews
- Fake ratings
- Unsupported prices
- Inaccurate address
- Unverified hours
- Overlapping schema
- Duplicate schema
- Missing required fields
- Conflicting schema types
- Schema that does not match visible content

For each risk, provide a prevention strategy.

## Part 9: Claude Code Implementation Notes

Provide:

- Existing schema pattern to inspect
- File types likely involved
- Where schema may belong
- JSON-LD notes
- Validation notes
- Data that must be confirmed before implementation
- Fields to leave out if not verified

## Part 10: Final Schema Plan Summary

End with:

1. Recommended schema types
2. Must-have schema
3. Optional schema
4. Schema to avoid
5. Data needed before implementation
6. Validation checklist
7. Claude Code build notes

Do not invent missing data.

Focus on accurate schema planning and implementation readiness.
